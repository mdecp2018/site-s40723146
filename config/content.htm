Git常用基本指令

git add .    新增所有改版內容

git commit -m "提交訊息, 用來辨識版本"

git push  在近端提交改版內容

git status  查驗狀態

cls 清除指令

git branch 顯示分支清單

git clone <url> 複製既有的遠端數據庫

Week1

如何利用 p37 中的 git 建立個人的 Github Pages 網站 

測試影片

1.進入github 登入帳號 開起p37 然後git clone (week1的2018fall)之內容

*網址https://github.com/mdecourse/2018fall+(資料夾名稱) 就可以將在此網站下載的資料存取在你所取名的資料夾

2.建立新的倉庫 New repository 填入所想要的資料夾名稱(Repository name) 並勾取 README

3.gitclone 建立的倉儲並將剛剛下載的2018fall內容 除了.git資料夾以外選取並複製然後貼上到自己建立的倉儲資料夾裡
*不複製.git是因為這是他人的版本資料，我們只需要內容就好

4.已經把2018fall的資料貼到自己的倉儲後 因為我們只有貼到近端 所以我們要從近端推送到遠端 做一個更新的動作
用p37進入自己的倉儲 cd (資料夾名稱) > 輸入 git status 檢查有無新增 > git add . 新增全部到近端資料庫

*因為我們要更新 系統必須知道我們是誰
所以要輸入帳號 git config --global user.email "(信箱帳號)"
還有使用者名稱 git config --global user.name "(使用者名稱)"

我們現在已經有身分 所以我們輸入 git commit -m "initial add" 這指令讓我們知道 我們在這次的改版改了什麼
再來我們要推上遠端 輸入git push 輸入帳號密碼後等待傳輸就會推上遠端

5.成功推上遠端後 我們想要從近端檢查 所以我們將資料夾裡的http-server.py用SciTe開啟
開啟之後 啟動(F5) 在網頁輸入 https://localhost:8444 點選前往後就可以查看
要從遠端檢查的話 我們輸入https://(資料夾名).github.io/(資料夾名)/content/index.html

6.那如果我們想要從近端改版的話 P37 輸入 python wsgi.py 後複製程式給你的網址 https://127.0.0.1:8443/
進到裡面後 登入 輸入密碼 admin 就可以登入進去
